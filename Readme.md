## 法律RAG智能体 · 详细设计Prompt v2.0

### 一、产品定位

一个面向法律实务场景的智能助手，核心能力是：
- 基于法律法规、司法解释、案例库进行精准检索与推理
- 支持三种推理范式切换（ReAct / Plan-and-Solve / Reflection）
- 具备完整的三层记忆与反思机制
- **支持用户自定义助手性格与说话方式**
- 所有敏感操作在沙箱内执行，并有严格输出护栏

---

### 二、自定义性格与说话方式

#### 2.1 可配置维度

在系统提示中预留以下可配置槽位，用户可通过设置面板或对话指令随时修改：

| 维度 | 可选值举例 | 说明 |
|------|-----------|------|
| `role` | 法律顾问 / 法务助理 / 学者型研究员 / 模拟对方律师 | 决定基本立场和专业深度 |
| `formality` | 正式严谨 / 半正式亲和 / 简明扼要 | 控制用语规范程度 |
| `verbosity` | 详细解释 / 标准篇幅 / 只给结论 | 控制输出长度 |
| `tone` | 冷静中立 / 坚定有力 / 温和耐心 | 控制情绪色彩 |
| `language` | 中文 / 英文 / 中英双语 | 输出语言 |
| `address_user` | 您 / 你 / 当事人 / 客户 | 对用户的称呼方式 |
| `conclusion_first` | true / false | 是否结论先行（适合忙碌的律师） |

#### 2.2 注入机制

这些配置在每轮对话开始时，以结构化JSON注入系统提示的`persona`区块：

```json
{
  "persona": {
    "role": "法务助理",
    "formality": "正式严谨",
    "verbosity": "标准篇幅",
    "tone": "冷静中立",
    "language": "中文",
    "address_user": "您",
    "conclusion_first": true
  }
}
```

系统提示中明确要求Agent：
- 严格遵守上述配置生成回复
- 当用户指令与性格配置冲突时，优先遵循用户最新指令
- 用户可以通过自然语言临时或永久修改这些配置（如"从现在开始用英文回答，简短一点"）

### 三、法律RAG核心能力

#### 3.1 知识库设计

**三层知识结构**：

1. **法规库**（法律、行政法规、司法解释、部门规章）
   - 每条记录包含：标题、发文机关、文号、颁布/生效日期、效力级别、全文（分段存储）、关键条款标签
   
2. **案例库**（指导性案例、典型案例、普通裁判文书）
   - 每条记录包含：案件名称、案号、审理法院、裁判日期、案由、争议焦点、裁判要旨、关联法条
   
3. **实务知识库**（常见法律问题解答、合同模板、风险清单）
   - 结构化QA对或文档片段

#### 3.2 检索增强生成流程

用户提问后，系统执行以下流水线：

```
用户Query
  │
  ├─→ [意图识别] 判断：法条查询 / 案例检索 / 实务咨询 / 合同审查 / 法律推理
  │
  ├─→ [查询改写] 将口语化问题改写为精确的法律检索query
  │     例："公司不给钱能不能告" → "用人单位拖欠劳动报酬 劳动仲裁 起诉条件"
  │
  ├─→ [多路召回]
  │     ├─ 向量检索（语义相似度 Top K1）
  │     ├─ 关键词检索（BM25 / 精确匹配 Top K2）
  │     └─ 法条关联检索（通过案由→法条映射补召回）
  │
  ├─→ [重排序] 使用法律领域微调的重排序模型，综合相关性、效力级别、时效性
  │
  ├─→ [上下文注入] 将Top N结果以结构化格式注入系统提示：
  │     - 法条原文 + 效力状态
  │     - 案例要旨 + 裁判结果
  │     - 注明来源与时效性警告（如"本法条已被XX修订，请核实"）
  │
  └─→ [生成回答]
        - 要求逐条引用来源
        - 区分"明确法律规定"与"实践倾向"
        - 置信度标注（见下文）
```

#### 3.3 法律专属约束

- **必须引用来源**：每个法律结论必须附引用（法规名称+条款号，或案例案号+裁判要旨）
- **时效性警告**：涉及已修订或被废止的法律时，必须明确提示
- **地域效力提醒**：地方法规、司法文件需注明适用范围
- **不构成法律意见声明**：每次回答末尾自动追加免责声明（用户可在性格配置中关闭）

#### 3.4 工具预加载（法律场景）

根据意图识别结果，预先注入对应工具文档：

| 意图 | 预加载工具 |
|------|-----------|
| 法条查询 | 法规检索API、法规时效性校验API、关联法条推荐API |
| 案例检索 | 案例检索API、案由分类API、法院层级过滤API |
| 合同审查 | 合同条款解析API、风险条款识别API、对比模板库API |
| 法律计算 | 诉讼费计算器、工伤赔偿计算器、诉讼时效计算器 |

### 四、三层记忆（法律场景适配）

#### 4.1 工作记忆
- 当前对话完整历史
- 注入的法规/案例片段
- 当前推理链

#### 4.2 短期记忆（跨轮关键事实）
存储当前会话周期内的：
- 用户身份信息（当事人/律师/法务，所在行业）
- 当前案件/项目的核心要素（案由、标的额、管辖地、关键时间节点）
- 用户偏好（倾向调解/诉讼、风险偏好）

#### 4.3 长期记忆
- **向量数据库**存储：
  - 成功的法律推理链路
  - 反思后的经验教训（如"XX案由下主张A比主张B胜诉率更高"）
  - 用户长期偏好
- **事件摘要库**存储：
  - 每次咨询的结构化摘要（咨询类型、涉及法条、最终建议、后续事项）

#### 4.4 主动记忆压缩（不变）
触发条件、摘要Agent指令、保留要素（实体/关系/未完成任务）与通用版一致。

### 五、反思、置信度与安全（法律场景增强）

#### 5.1 置信度自评估
法律场景下置信度阈值设定更严格：

| 置信度区间 | 行为 |
|-----------|------|
| ≥0.9 | 直接输出，引用法条原文支撑 |
| 0.7-0.9 | 输出但标注"需进一步核实"并提供核实路径 |
| 0.5-0.7 | 暂停，告知用户存在的模糊地带，询问是否补充信息或缩小范围 |
| <0.5 | 拒绝给出明确意见，说明不确定原因，建议咨询专业律师 |

#### 5.2 反思链（法律场景）
每个任务完成后，反思Agent侧重检查：
- 法条引用是否准确、完整
- 是否遗漏例外条款或但书
- 案例与当前事实的相似度是否足够
- 是否注意到新法优于旧法、特别法优于一般法等原则
- 地域管辖、时效等程序性问题是否考虑

反思结果以结构化教训存入长期记忆，格式：
```json
{
  "task_type": "劳动争议-违法解除赔偿",
  "lesson": "主张2N赔偿时需同时准备证明劳动关系的证据和单位违法解除的证据，缺一不可",
  "source_case": "用户咨询ID: xxx",
  "reusability_score": 0.85
}
```

#### 5.3 输出护栏（法律场景增强）
安全过滤器增加法律特有规则：
- **禁止输出**：伪造法条、编造案号、对未公开裁判结果的确切预测
- **警告拦截**：涉及刑事犯罪、危害国家安全、教唆伪造证据等高风险话题时，直接拦截并提示
- **敏感脱敏**：自动检测并脱敏身份证号、手机号、住址等个人信息

#### 5.4 思维回溯（法律场景）
遇到推理错误时，回溯Agent特别关注：
- 是否错误适用了已废止的法条
- 是否忽略了上位法/特别法的优先性
- 是否混淆了相似案由（如"劳务关系"与"劳动关系"）
- 是否在案例类比时忽略了关键事实差异

### 六、实现优先级建议

**MVP阶段（第一轮交付）**：
1. 法律知识库搭建 + 基础RAG检索
2. 三种推理范式手动切换
3. 自定义性格配置
4. 三层记忆框架（重点：工作记忆+短期记忆）
5. 强制引用来源

**第二轮迭代**：
1. 主动记忆压缩
2. 置信度自评估
3. 输出护栏（规则引擎部分）

**第三轮迭代**：
1. 事后反思链 + 长期记忆存储
2. 思维回溯
3. 工具预加载与上下文注入
4. 沙箱包装器
